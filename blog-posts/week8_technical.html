What is a web application?

You know those applications and programs you use all the time? Like Microsoft Word before there was such thing as Google Docs? No? Um, well, how about videogames? That's what you would call software. Programs that are executed using computers. Another word for those programs? Applications! So take a guess, what's a WEB application then?

I'm going to assume you guessed correctly, because you're awesome.

A web application is ANY piece of software that runs inside of a web browser, like Google Chrome or Firefox or Safari. Some notable examples would be Google Docs, Facebook, Twitter...I could go on. Web applications are huge nowadays, and do all sorts of stuff. One of my personal favorites is the Mint.com web app!

Ok let's move on.

What is a framework?

In this case, we're talking about web application frameworks, and what those do: to make a long story short? They make it way easier to develop web applications. Hmm...what's a good metaphor to describe a framework....OH, I GOT ONE.

Let's say you make hamburgers. And someone comes up with a framework for making those delicious burgers. All you have to do, is get and cook the meat, the framework will handle the organization of the bun and toppings.

Well, maybe not the best way to describe a complex system. But I'm trying damn it!

The cool thing about frameworks is they have tools to help pull together everything you need to make a web application. From integrating HTML/CSS/JavaScript code, and creating simple ways to interface with your app's databases and SQL.

What is Rails? Why Should I Care?

Rails is one of the hottest ways to make a web application right now in the world of developers. It's a web application framework that uses the model-view-controller pattern AKA MVC. I'm going to break that down real fast: It's a pattern that divides application activities into 3 parts.

  • The Model: Manages the data, logic, and rules of the application
  • The View: Basically, the output of the application, what a user sees based on information it's been given (think about View options on Sublime or Document applications, do you want landscape view? Sort of like that..)
  • The Controller: The part that accepts input, and allows you to affect and manipulate the Model. Basically, think COMMANDING/CONTROLLING the MODEL to change your VIEW to a 2 column view on your Word Editor

Ok, back to Rails! It was developed by a dude named David Heinemeier Hansson who really dug Ruby and thought it'd be perfect for his idea to create a streamlined framework for developing on the Web. I'd say he did pretty well for himself given how much people talk about Rails.

Here's why you should care: If you know how to use it well. People will pay you, very well. A little mercenary of me to flat out say...but as far as I've seen, it's true. Apparently's there's a high demand for skilled Ruby/Ruby on Rails develoeprs. So happy hunting you aspiring developer!!
